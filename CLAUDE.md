# GTA 6 Asset Library — правила для Claude

Это библиотека медиа-материалов (картинки, скриншоты, логотипы, арты) для создания видеорядов про GTA 6. Claude сам модерирует репозиторий по этим правилам.

## Структура

```
characters/    персонажи (Jason, Lucia, NPC) — подпапка на персонажа
locations/     локации (Vice City, Leonida Keys...) — подпапка на локацию
vehicles/      транспорт
logos/         логотипы (Rockstar, GTA VI, внутриигровые бренды)
screenshots/   скриншоты из трейлеров/геймплея
artwork/       официальные арты, постеры, ключевые арты
maps/          карты и фрагменты карты
ui/            интерфейс, шрифты, HUD
misc/          всё, что не подошло выше
```

Подпапки создавай по мере накопления: если в папке 3+ файла про одну сущность — выноси в подпапку (`characters/lucia/`).

## Правила для каждого ассета

1. **Имя файла** — человекочитаемое, snake_case, на английском: `lucia_beach_trailer1_01.jpg`, не `IMG_2041.png`.
2. **Sidecar-описание** — рядом с каждым файлом `имя.md` с фронтматтером:

```markdown
---
title: Lucia on the beach
source: GTA VI Trailer 1 (timestamp 0:43)
type: screenshot
tags: [lucia, beach, vice-city, day]
resolution: 1920x1080
license: Rockstar promo / fair use
added: 2026-07-23
---

Люсия стоит на пляже Vice City, закатное освещение. Хорошо подходит
для сцен про главных героев и атмосферу города.
```

3. **INDEX.md в корне каждой папки** — таблица: файл → одна строка описания. Обновляй при каждом добавлении/удалении.
4. **Корневой INDEX.md** — дерево всей библиотеки. Это первое, что читаешь при поиске ассета.

## Модерация (делай сам при каждой работе с репо)

- Новый файл без описания → создай sidecar `.md` (посмотри картинку через Read и опиши её).
- Файл с мусорным именем → переименуй по конвенции.
- Дубликаты → удали худший по качеству.
- После изменений → обнови INDEX.md и закоммить с осмысленным сообщением.

## Поиск ассетов

При задаче «найди картинку X для видео»: сначала `INDEX.md` корня → INDEX нужной папки → фронтматтер-теги (`grep -r "tags:.*слово" --include="*.md"`).
